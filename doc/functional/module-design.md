# 功能模块详细设计

## 1. 模块总览

```
绩效管理系统
├── 首页/看板（Dashboard）
├── 绩效管理
│   ├── 绩效计划管理
│   ├── 指标管理
│   ├── 进度跟踪

│   └── 绩效校准
├── 数据分析
│   ├── 绩效报表
│   ├── 趋势分析
│   └── 对比分析
├── 组织管理

```

## 2. 首页/看板模块（Dashboard）

### 2.1 功能定位
为不同角色提供个性化的绩效概览和待办事项

### 2.2 页面布局

#### 布局结构
```
┌─────────────────────────────────────────┐
│  周期选择器 | 刷新按钮                    │
├─────────────────────────────────────────┤
│  [概览卡片1] [概览卡片2] [概览卡片3] ... │
├─────────────────────────────────────────┤
│  [图表区：趋势图]  [图表区：分布图]      │
├─────────────────────────────────────────┤
│  [待办事项]        [风险预警]            │
├─────────────────────────────────────────┤
│  [部门排名]        [Top Performer]      │
└─────────────────────────────────────────┘
```

### 2.3 功能点详细设计

#### 2.3.1 概览卡片

**员工视角**：
- 当前周期
- 个人完成率
- 当前分数
- 距离截止日期天数

**主管视角**：
- 团队人数
- 团队完成率
- 团队平均分
- 待评估人数

**HR/管理员视角**：
- 全公司参与人数
- 整体完成率
- 平均分
- 风险人数

**交互**：
- 点击卡片跳转到对应详情页
- 悬停显示详细说明

---

#### 2.3.2 趋势图

**图表类型**：折线图

**数据维度**：
- X 轴：时间（最近 5 个周期）
- Y 轴：平均分数

**筛选条件**：
- 按部门筛选
- 按岗位筛选

**交互**：
- 点击数据点查看详情
- 图例切换显示/隐藏

---

#### 2.3.3 分布图

**图表类型**：饼图/柱状图

**数据维度**：
- A/B/C/D 等级分布
- 各等级人数和占比

**交互**：
- 点击扇区查看该等级人员列表
- 悬停显示具体数值

---

#### 2.3.4 待办事项

**列表内容**：
- 待提交绩效计划
- 待审批绩效计划
- 待自评
- 待上级评价
- 待校准

**每条待办包含**：
- 标题
- 截止时间
- 优先级（高/中/低）
- 操作按钮（立即处理）

**交互**：
- 点击跳转到对应页面
- 标记为已读
- 批量处理

---

#### 2.3.5 风险预警

**预警类型**：
- 进度滞后（进度 < 60% 且时间 > 70%）
- 分数偏低（< 70 分）
- 未提交（截止前 3 天）
- 未评估（截止前 2 天）

**展示方式**：
- 红色警示图标
- 风险描述
- 建议措施

**交互**：
- 点击查看风险详情
- 一键发送提醒

---

#### 2.3.6 部门排名

**排序规则**：
- 按平均分降序
- 显示前 10 名

**展示字段**：
- 排名
- 部门名称
- 平均分
- 人数
- 较上期变化（↑↓）

---

#### 2.3.7 Top Performer

**筛选规则**：
- 分数最高的前 5 名
- 等级为 A

**展示字段**：
- 头像
- 姓名
- 部门
- 分数
- 等级

---

## 3. 绩效计划管理模块

### 3.1 功能定位
管理绩效计划的整个生命周期：创建 → 提交 → 审批 → 执行 → 评估

### 3.2 页面列表

#### 3.2.1 计划列表页

**筛选条件**：
- 绩效周期（必选）
- 部门
- 状态
- 员工姓名/工号
- 等级

**列表字段**：
| 字段 | 说明 | 可排序 |
|------|------|--------|
| 员工姓名 |  clickable | 是 |
| 工号 | - | 是 |
| 部门 | - | 是 |
| 周期 | - | 否 |
| 状态 | 标签展示 | 是 |
| 进度 | 进度条 | 是 |
| 总分 | - | 是 |
| 等级 | 彩色标签 | 是 |
| 提交时间 | - | 是 |
| 操作 | 按钮组 | - |

**操作按钮**：
- 查看（所有状态）
- 编辑（仅草稿）
- 提交（仅草稿）
- 审批（仅待审批）
- 删除（仅草稿）

**批量操作**：
- 批量导出
- 批量提醒

---

#### 3.2.2 计划详情页

**Tab 结构**：
```
基本信息 | 指标列表 | 执行记录 | 评分记录 | 审批流程 | 面谈记录
```

**① 基本信息 Tab**
- 员工信息（姓名、工号、部门、岗位）
- 周期信息
- 状态
- 总分
- 等级
- 评估人
- 关键时间点（创建、提交、审批、评估、校准）

**② 指标列表 Tab**

表格展示：
| 指标名称 | 类型 | 权重 | 目标值 | 当前值 | 进度 | 得分 | 状态 |
|---------|------|------|--------|--------|------|------|------|

- 进度用进度条展示
- 状态用标签展示
- 点击可查看指标详情

**③ 执行记录 Tab**

时间轴展示：
- 周报/月报
- 里程碑
- 成果记录

每条记录包含：
- 日期
- 类型
- 内容
- 进度
- 附件
- AI 总结（如有）

**④ 评分记录 Tab**

展示所有评分：
- 自评分
- 上级评分
- 同事评分（360）
- 下级评分（360）

每个评分包含：
- 评分人
- 评分类型
- 分数
- 评语
- 维度评分详情
- 评分时间

**⑤ 审批流程 Tab**

流程图展示：
```
创建 → 提交 → 审批 → 执行中 → 评估 → 校准 → 归档
```

每个节点显示：
- 操作人
- 操作时间
- 意见

**⑥ 面谈记录 Tab**

列表展示历次面谈：
- 面谈日期
- 面谈人
- 内容
- 改进计划
- 跟进日期
- 状态

---

#### 3.2.3 创建/编辑计划页

**步骤条**：
```
选择员工 → 添加指标 → 确认提交
```

**步骤 1：选择员工**
- 下拉选择或搜索
- 显示员工基本信息
- 检查是否已有该周期的计划

**步骤 2：添加指标**

两种方式：
1. **从指标库选择**
   - 搜索指标
   - 勾选添加
   - 设置目标值和权重

2. **自定义指标**
   - 输入指标名称
   - 选择类型（KPI/OKR）
   - 设置权重、目标值、单位
   - 选择计算方式（自动/手动）

**权重校验**：
- 实时显示权重总和
- 必须等于 100% 才能提交
- 超出或不足时红色提示

**步骤 3：确认提交**
- 预览所有指标
- 确认无误后提交
- 提交后进入审批流程

---

### 3.3 业务规则

#### 3.3.1 状态流转规则

| 当前状态 | 可执行操作 | 下一状态 | 权限要求 |
|---------|-----------|---------|---------|
| 草稿 | 编辑、提交、删除 | 待审批 | 员工本人 |
| 待审批 | 审批通过、驳回 | 执行中/草稿 | 主管 |
| 执行中 | 更新进度、添加记录 | - | 员工本人 |
| 执行中 | 开始评估 | 待评估 | 系统自动 |
| 待评估 | 自评 | - | 员工本人 |
| 待评估 | 上级评分 | 已评估 | 主管 |
| 已评估 | 校准 | 已校准 | HR/管理员 |
| 已校准 | 归档 | 已归档 | 系统自动 |

#### 3.3.2 权限控制

- **员工**：只能查看和操作自己的计划
- **主管**：可以查看团队成员的计划，进行审批和评分
- **HR**：可以查看全公司计划，执行校准
- **管理员**：所有权限

## 4. 指标管理模块

### 4.1 功能定位
建立统一的指标库，支持指标的复用和标准化

### 4.2 页面列表

#### 4.2.1 指标列表页

**视图切换**：
- 列表视图
- 树形视图（按分类）

**筛选条件**：
- 指标类型（KPI/OKR/BSC）
- 分类（财务/客户/内部/学习）
- 计算方式（自动/手动）
- 是否启用

**列表字段**：
| 字段 | 说明 |
|------|------|
| 指标名称 | clickable |
| 类型 | 标签 |
| 分类 | 标签 |
| 默认权重 | - |
| 计算方式 | - |
| 数据来源 | - |
| 使用次数 | 被引用次数 |
| 状态 | 启用/禁用 |
| 创建时间 | - |
| 操作 | 编辑、禁用、删除 |

**操作**：
- 新建指标
- 批量导入（Excel）
- 批量导出
- 批量启用/禁用

---

#### 4.2.2 指标详情/编辑页

**表单字段**：

**基本信息**：
- 指标名称（必填）
- 指标类型（必填，单选：KPI/OKR/BSC）
- 指标分类（可选，单选）
- 描述（多行文本）

**计算规则**：
- 计算方式（必填，单选：自动/手动）
- 计算公式（自动计算时填写）
- 数据来源（自动计算时选择：API/MES/ERP/手动）
- API 地址（如选择 API）
- 数据同步频率（每日/每周/每月）

**默认配置**：
- 默认权重（0-100）
- 单位（元/个/%/分等）
- 目标值参考范围

**其他**：
- 父指标（支持层级结构）
- 是否启用
- 排序

---

#### 4.2.3 指标分类管理

**树形结构管理**：
- 添加分类
- 编辑分类
- 删除分类（需检查是否有子分类或指标）
- 拖拽调整顺序

**默认分类**：
- 财务维度
- 客户维度
- 内部流程维度
- 学习与成长维度

---

### 4.3 业务规则

- 指标被引用后不可删除，只能禁用
- 指标名称在同一分类下唯一
- 自动计算的指标需配置数据源
- 指标变更需记录版本历史

---

记录绩效执行过程，实时跟踪进度

### 5.2 页面列表

**我的指标卡片**：

```
┌─────────────────────────────┐
```

**快速操作**：
- 填报周报
- 上传成果
- 记录里程碑

---

#### 5.2.2 周报/月报填报

**表单字段**：
- 报告类型（周报/月报）
- 报告期间（自动填充）
- 本周/本月工作内容（富文本编辑器）

- 遇到的问题和风险

**AI 辅助**：
- 自动生成工作总结
- 提取关键成果
- 识别风险点

**提交**：
- 保存草稿
- 提交（通知主管）

---

#### 5.2.3 团队进度页（主管视角）

**团队成员进度概览**：

表格展示：
| 员工 | 指标数 | 平均进度 | 风险数 | 最后更新 | 操作 |
|------|--------|---------|--------|---------|------|

**风险标识**：
- 红色：进度严重滞后
- 黄色：进度轻微滞后
- 绿色：正常

**操作**：
- 查看详情
- 发送提醒
- 批量查看周报

---

#### 5.2.4 进度更新弹窗

**触发方式**：
- 点击"更新进度"按钮

**表单字段**：
- 当前值（数字输入）
- 进度百分比（自动计算或手动调整）
- 状态（未开始/进行中/已完成/延期）
- 备注说明

**历史记录**：
- 显示历次更新记录
- 时间轴展示

---

### 5.3 业务规则

- 周报需在每周五下班前提交
- 月报需在每月最后一天提交
- 进度更新需记录历史
- 自动计算的指标从外部系统同步，不可手动修改
- 进度滞后时自动发送预警

## 6. 绩效评估模块

### 6.1 功能定位
多维度绩效评价，支持自评、上级评、360 评价

### 6.2 页面列表

#### 6.2.1 待评估列表

**Tab 切换**：
- 待自评
- 待我评价（上级）
- 待同事评价（360）

**列表字段**：
| 员工 | 部门 | 周期 | 自评状态 | 上级评分 | 截止日期 | 操作 |
|------|------|------|---------|---------|---------|------|

**筛选**：
- 部门
- 截止日期
- 评分状态

---

#### 6.2.2 评分页面

**页面布局**：
```
┌──────────────────────────────────────┐
│ 员工信息卡片                          │
├──────────────────────────────────────┤
│ 指标评分表                            │
│ ┌──────┬────┬────┬────┬────┐        │
│ │指标  │权重│目标│实际│得分│        │
│ ├──────┼────┼────┼────┼────┤        │
│ │...   │    │    │    │    │        │
│ └──────┴────┴────┴────┴────┘        │
├──────────────────────────────────────┤
│ 维度评分                              │
│ ○ 工作质量  [★★★★☆]  90分           │
│ ○ 工作效率  [★★★★☆]  85分           │
│ ○ 团队协作  [★★★★★]  95分           │
├──────────────────────────────────────┤
│ 评语                                  │
│ [多行文本框]                           │
├──────────────────────────────────────┤
│ AI 评分建议: 85分                     │
│ [采纳建议] [忽略]                      │
├──────────────────────────────────────┤
│          [保存草稿]  [提交评分]        │
└──────────────────────────────────────┘
```

**评分方式**：
1. **指标评分**
   - 自动计算的指标：系统自动填充
   - 手动评分的指标：输入分数（0-100）

2. **维度评分**
   - 星级评分（1-5 星）
   - 或滑块评分（0-100）
   - 每个维度可填写评语

3. **综合评分**
   - 系统自动计算加权分
   - 可手动调整（需填写原因）

**AI 辅助**：
- 显示 AI 建议分数
- 显示评分理由
- 一键采纳建议

**提交校验**：
- 所有必填项已完成
- 分数在合理范围内
- 评语不能为空

---

#### 6.2.3 360 评价邀请

**邀请流程**：
1. 选择被评价人
2. 选择评价人（同事、下级）
3. 设置截止时间
4. 发送邀请邮件/站内信

**评价人选择**：
- 自动推荐（同项目组、协作频繁）
- 手动添加
- 至少 3 人评价

**匿名设置**：
- 可选择匿名评价
- 被评价人看不到具体评价人

---

### 6.3 业务规则

- 自评必须在上级评分之前完成
- 上级评分完成后才能进行 360 评价
- 360 评价至少需要 3 人完成
- 评分提交后不可修改（特殊情况需 HR 解锁）
- 评分截止时间后自动锁定

---

## 7. 绩效校准模块

### 7.1 功能定位
确保评分公平性，符合强制分布要求

### 7.2 页面列表

#### 7.2.1 校准列表页

**筛选条件**：
- 绩效周期
- 部门
- 当前等级

**列表字段**：
| 员工 | 部门 | 原分数 | 原等级 | 校准分数 | 校准等级 | 校准原因 | 操作 |
|------|------|--------|--------|---------|---------|---------|------|

**批量操作**：
- 批量校准
- 导出校准记录

---

#### 7.2.2 校准工作台

**页面布局**：
```
┌────────────────────────────────────────┐
│ 分布控制面板                            │
│ A级: 15人 (15%) [=====] 目标: 20%     │
│ B级: 60人 (60%) [===========] 目标: 70%│
│ C级: 20人 (20%) [====] 目标: 10%      │
│ D级: 5人 (5%)  [=] 目标: 0%           │
├────────────────────────────────────────┤
│ 人员拖拽区                              │
│ ┌─────┐ ┌─────┐ ┌─────┐              │
│ │A级  │ │B级  │ │C级  │              │
│ │张三 │ │李四 │ │王五 │              │
│ │95分 │ │85分 │ │75分 │              │
│ └─────┘ └─────┘ └─────┘              │
│ 拖拽人员调整等级                        │
├────────────────────────────────────────┤
│ 校准建议列表                            │
│ ○ 张三: 建议从 B 调到 A                │
│   原因: 分数88分，接近A级线             │
│   [采纳] [忽略]                         │
└────────────────────────────────────────┘
```

**功能点**：

1. **分布控制**
   - 显示当前分布比例
   - 显示目标分布比例
   - 实时计算偏差

2. **拖拽调整**
   - 拖拽员工卡片到不同等级区域
   - 自动更新分数和等级
   - 必须填写调整原因

3. **智能建议**
   - AI 分析边界人员
   - 给出调整建议
